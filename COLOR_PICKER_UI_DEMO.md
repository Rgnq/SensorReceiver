## 交互流程演示

### 场景1: 修改背景色

```
Step 1: 用户看到SettingsPage
        │
        ├─ 确认编辑主题为"深色主题"
        └─ 看到6个颜色按钮及其当前值

Step 2: 用户点击"背景色"按钮 (■ 按钮)
        │
        └─ QColorDialog 调色盘打开
            ┌─────────────────────┐
            │   选择颜色对话框     │
            │ ┌────────────────┐  │
            │ │  调色板区域    │  │
            │ │  ┌──────────┐  │  │
            │ │  │  ■■■     │  │  │
            │ │  │  ■■■     │  │  │
            │ │  └──────────┘  │  │
            │ └────────────────┘  │
            │  当前色: #121212    │
            │  新色: #1A3A2E      │
            │ [确定] [取消]      │
            └─────────────────────┘

Step 3: 用户选择新颜色 #1A3A2E
        │
        └─ 点击[确定]按钮

Step 4: 后台处理
        │
        ├─ 更新 dark_theme_colors['Background'] = '#1A3A2E'
        ├─ 保存到 config/dark_theme_colors.json
        ├─ 发出 themeColorsChangedSignal(True)
        └─ 调用 MainWindow.on_theme_colors_changed()

Step 5: UI自动更新
        │
        ├─ 背景色按钮显示新颜色 ■ (深绿色)
        ├─ 颜色值标签更新 #1A3A2E
        ├─ 整个应用背景立即变为深绿色
        └─ 所有相关组件自动刷新
```

### 场景3: 重置颜色

```
用户点击 [重置为默认颜色] 按钮
            ↓
确认对话框弹出
    "确定要重置所有颜色为默认值吗?"
            ↓

```

## 数据流动图

```
用户选择颜色
    │
    ↓
QColorDialog.getColor() 返回新颜色
    │
    ↓
保存到 dark_theme_colors[key] = hex_color
    │
    ↓
调用 save_theme_colors(is_dark, colors)
    │
    ├─→ 创建 config/ 目录 (如果不存在)
    │
    └─→ 写入 JSON 文件
        {
            "Background": "#121212",
            "Surface": "#1E1E1E",
            ...修改的颜色值...
        }
    │
    ↓
发出 themeColorsChangedSignal(is_dark)
    │
    ↓
MainWindow.on_theme_colors_changed(is_dark)
    │
    ├─→ 调用 MainWindow.update_style(is_dark)
    │
    └─→ 调用各组件的 update_style(is_dark)
        │
        ├─→ menubar.update_style(is_dark)
        ├─→ sidebar.update_style(is_dark)
        ├─→ Homepage.update_style(is_dark)
        └─→ ...其他页面.update_style(is_dark)
    │
    ↓
所有组件重新获取颜色值
    │
    ├─→ colors = get_theme_colors(is_dark)
    └─→ 应用新的样式表
    │
    ↓
UI 实时刷新,应用立即显示新颜色
```

## 配置文件示例

### config/dark_theme_colors.json
```json
{
    "Background": "#121212",
    "Surface": "#1E1E1E",
    "Text_Primary": "#E0E0E0",
    "Text_Secondary": "#A0A0A0",
    "Accent": "#BB86FC",
    "Border_Hover": "#333333"
}
```

### config/light_theme_colors.json
```json
{
    "Background": "#FFFFFF",
    "Surface": "#C8C8C8",
    "Text_Primary": "#000000",
    "Text_Secondary": "#606060",
    "Accent": "#6200EE",
    "Border_Hover": "#CCCCCC"
}
```

| 操作 | 响应时间 |
|-----|--------|
| 打开调色盘 | <100ms |
| 保存颜色到文件 | ~50ms |
| UI更新 | <200ms (实时) |
| 切换主题 | <150ms |
| 重置颜色 | ~100ms |

## 注意事项

1. **首次使用** - config目录会自动创建
2. **颜色格式** - 必须是有效的HEX格式 (#RRGGBB)
3. **深浅独立** - 两个主题的颜色完全独立
4. **实时保存** - 每次修改都立即保存到JSON
5. **文件损坏** - 如果JSON损坏,应用会使用默认颜色
