# SensorReceiver 多语言本地化功能更新说明

## 更新内容概述

本次更新为 SensorReceiver 应用程序添加了完整的多语言支持系统（i18n），并在设置页面中增加了语言选择功能。

## 新增功能

### 1. 多语言支持系统 (i18n)
- **新文件**: `i18n.py` - 国际化核心模块
- **翻译文件**:

  - `i18n/en_US.json` - 英文翻译
- **语言偏好保存**: `config/language.json` - 用户选择的语言持久化存储

### 2. 设置页面增强

- 下拉菜单选择语言（简体中文/English）
- 实时更新所有UI文本

### 3. UI 文本国际化

- **Homepage** (首页/首ページ)
- **HistoryPage** (历史/履歴)
- **SettingsPage** (设置/設定)
- **LogPage** (日志/ログ)
- **CommandPanel** (命令面板)

### 4. 语言切换信号
- 新增 `languageChangedSignal` 信号
- 支持动态更新所有UI组件文本
- 每个页面都实现了 `update_ui_text()` 方法

## 使用方法

### 用户操作
1. 打开应用程序的"设置"页面
2. 在"语言"下拉菜单中选择所需语言
3. UI文本立即更新为选定语言

### 开发者集成
在代码中使用翻译：
```python
from i18n import t

# 简单使用
label.setText(t("page.homepage.title"))

# 在UI更新方法中
def update_ui_text(self):
    self.button.setText(t("page.settings.apply"))
```

## 常见问题

### Q: 如何添加新的翻译？
A: 编辑 `i18n/zh_CN.json` 和 `i18n/en_US.json`，在相应位置添加新的键值对，然后在代码中使用 `t("path.to.key")`。

### Q: 如何添加新的语言？
A: 
1. 在 `i18n/` 目录中创建新的JSON文件（如 `ja_JP.json`）
2. 在 `i18n.py` 的 `_supported_languages` 字典中注册
3. 翻译所有文本

### Q: 语言改变后需要重启吗？
A: 大多数UI文本会立即更新，但某些组件可能需要重启应用以应用所有更改。

## 已知限制

1. 某些由 qt_material 库生成的动态文本（如样式名称）不支持翻译
2. 错误消息中的某些动态内容可能需要手动处理
